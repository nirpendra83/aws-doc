Q1. Manage Identity Providers

  • Configure OAuth to use HTPasswd as the identity provider.
  • Identity Provider name: ex280-provider
  • Create users: harry, leader, raja, qa-engineer with password review
  • Create secret named super-secret to store credentials

Solution:

# Create HTPasswd file with first user (creates file)
htpasswd -c -B -b /home/student/htpasswd harry review

# Add other users to the same file
htpasswd -b /home/student/htpasswd leader review
htpasswd -b /home/student/htpasswd raja review
htpasswd -b /home/student/htpasswd qa-engineer review

# Verify htpasswd contents
cat /home/student/htpasswd

# Create secret in openshift-config namespace with htpasswd file
oc create secret generic super-secret --from-file=htpasswd=/home/student/htpasswd -n openshift-config

# Export current OAuth config to yaml
oc get oauth cluster -o yaml > oauth.yaml

# Edit oauth.yaml and add the following under spec.identityProviders:
# - name: ex280-provider
#   mappingMethod: claim
#   type: HTPasswd
#   htpasswd:
#     fileData:
#       name: super-secret

# Replace OAuth configuration with updated file
oc replace -f oauth.yaml

# Watch authentication pods to confirm reload
watch oc get pods -n openshift-authentication

# (Optional) Get OpenShift Web Console URL for login testing
oc whoami --show-console
