Gitlab on AWS Using Gitlab Environment Toolkit

  • Public Innersource Staging Environment
  • Raise a new request to DISI team for route 53 namesaerver update
https://soprasteriacorp.service-now.com/sp?id=ticket&table=sc_req_item&sys_id=0a3745ca478b6ed0254ce471026d4305&view=sp
  • Create a bastion host in the same network where you want to create your VMS so that there is communication to all the vms from bastion host
  • create a dir on vm
mkdir src
cd src
  • Clone the repo
git clone https://innersource.soprasteria.com/ops/gitlab/aws/instances/public-innersource-stg.git

  • Check the terraform & Ansible dirs

  • Set the environemt variable for AWS credentials or you can use IAM role to attach to a bastion host.

export AWS_ACCESS_KEY_ID=your_access_key_here
export AWS_SECRET_ACCESS_KEY=your_secret_key_here
export AWS_DEFAULT_REGION=eu-west-3
  • Create a toolkit container
podman run  -v ~/src/public-innersource-stg/ansible/environments:/gitlab-environment-toolkit/ansible/environments:Z   -v ~/src/public-innersource-stg/ansible/ansible.cfg:/gitlab-environment-toolkit/ansible/ansible.cfg:Z  -v ~/src/public-innersource-stg/ansible/roles:/gitlab-environment-toolkit/ansible/roles:Z     -v ~/src/public-innersource-stg/terraform/environments:/gitlab-environment-toolkit/terraform/environments:Z     -v ~/src/public-innersource-stg/terraform/modules/gitlab_ref_arch_aws:/gitlab-environment-toolkit/terraform/modules/gitlab_ref_arch_aws:Z    -v ~/src/public-innersource-stg/terraform/modules/gitlab_aws_instance:/gitlab-environment-toolkit/terraform/modules/gitlab_aws_instance:Z     -v ~/src/public-innersource-stg/keys:/gitlab-environment-toolkit/keys:Z     --name=toolkit -it registry.gitlab.com/gitlab-org/gitlab-environment-toolkit:3.7.1
  • Install required tools on toolkit container
pip install boto3 botocore
ansible-galaxy collection install amazon.aws
ansible-galaxy collection install amazon.aws --force

  • Check the Ansible inventory
