ðŸš€ Tanzu Kubernetes Grid (TKGs) Cluster Upgrade Guide

This guide outlines the steps to upgrade a Tanzu Kubernetes Cluster (TKC) running in a vSphere with Tanzu environment using kubectl.

1. Log in to the Supervisor Cluster:

kubectl vsphere login
–server=api-stg.tanzu.cloudonprem2.soprasteria.com
–insecure-skip-tls-verify=true
–vsphere-username nirkumar.ADM@emea.msad.sopra
–tanzu-kubernetes-cluster-namespace minio-stg
–tanzu-kubernetes-cluster-name minio-tkc

kubectl config use-context minio-stg

2. Update the TKC to a New Kubernetes Version:

kubectl edit tanzukubernetescluster minio-tkc -n minio-stg

Within the editor, modify the Kubernetes version as shown:

spec: distribution: version: v1.28.5+vmware.1

Ensure you select a version listed in the vSphere UI under:
Workload Management â†’ Clusters â†’ Supervisor Cluster â†’ Available Versions

3. Monitor the Upgrade Process:

kubectl get tanzukubernetescluster minio-tkc -n minio-stg -w kubectl get nodes

4. Troubleshooting:

If you encounter an Unauthorized error:

error: You must be logged in to the server (Unauthorized)

  • Switch to the Supervisor cluster context:

kubectl config use-context api-stg.tanzu.cloudonprem2.soprasteria.com

  • Verify your RBAC permissions or consult your administrator to gain access to:

kubectl get tanzukubernetesreleases

5. Additional Tips:

  • To view available Kubernetes versions without CLI access, navigate to:

    Workload Management â†’ Clusters â†’ Supervisor Cluster â†’ Available Versions

  • Always verify compatibility of the target version prior to upgrading.
