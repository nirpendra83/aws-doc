OpenShift Virtualization

OpenShift Virtualization (formerly Container-native Virtualization or CNV) is a feature of Red Hat OpenShift that allows users to run and manage virtual machines (VMs) alongside container workloads on the same platform. It integrates KubeVirt into OpenShift to provide a unified platform for both VMs and containers.

üß† Core Concepts & Terminologies

1. KubeVirt

An open-source project that extends Kubernetes by adding support for managing virtual machines as first-class citizens.

2. Virtual Machine (VM)

A software emulation of a physical computer that runs an operating system and applications. In OpenShift Virtualization, VMs are treated like Kubernetes objects.

3. VirtualMachine (VM) CRD

A Custom Resource Definition used to define VM objects within Kubernetes/OpenShift. You use this to declare VM specifications such as CPU, memory, disks, and more.

4. VirtualMachineInstance (VMI)

The runtime representation of a VM. When a VirtualMachine is started, a VirtualMachineInstance is created.

5. DataVolume (DV)

A Kubernetes custom resource used with Containerized Data Importer (CDI) to manage VM disk images. It simplifies provisioning persistent storage for VMs.

6. Containerized Data Importer (CDI)

A tool that facilitates importing, uploading, or cloning VM disk images into OpenShift Persistent Volume Claims (PVCs).

7. PersistentVolumeClaim (PVC)

Storage resource used by a VM’s disk. PVCs are dynamically provisioned and backed by supported storage classes.

8. Template

A reusable configuration that defines one or more OpenShift objects, such as a pre-configured VM, which can be instantiated multiple times.

9. virtctl

A CLI tool that provides convenience commands for managing virtual machines (e.g., start/stop/restart console access).

10. Cloud-Init

A widely used tool for customizing VM initialization (e.g., setting up users, SSH keys, packages) during boot time, often used with Linux VMs.

11. Node Placement

Defines scheduling rules for where VMs can be placed on nodes, often based on labels, affinities, or taints.

12. Live Migration

The ability to move a running VM from one node to another without downtime.

‚úÖ Benefits

  • Unified platform for containers and VMs
  • Simplified management with OpenShift UI and CLI
  • Seamless VM lifecycle integration into Kubernetes
  • Migration path from legacy hypervisors (e.g., VMware, Hyper-V)
  • Developer self-service for VMs

üîß kubevirt-hyperconverged: What Is It?

kubevirt-hyperconverged is the main operator used to deploy and manage OpenShift Virtualization on OpenShift clusters. It bundles all the required components to enable virtualization capabilities and simplifies the lifecycle management of the virtualization stack.

üì¶ Purpose

The kubevirt-hyperconverged operator acts as a meta-operator that installs and manages a set of other Kubernetes Operators and components necessary to support VMs in OpenShift, including:

  • KubeVirt (VM runtime)
  • CDI (Containerized Data Importer) (disk image management)
  • SSP (Scheduling, Scale and Performance)
  • Network Addons (e.g., multus, sriov)
  • Node Labeller
  • Template Validator
  • VM Templates

üìÅ Custom Resource (HCO CR)

The main Custom Resource managed by this operator is:

apiVersion: hco.kubevirt.io/v1beta1
kind: HyperConverged
metadata:
  name: kubevirt-hyperconverged
