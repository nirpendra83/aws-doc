VMware Tanzu Terms

Tanzu Kubernetes Concepts: TKG, TKR, TKC, and Cluster Types

This document provides a complete overview of the key concepts and cluster types in the VMware Tanzu Kubernetes Grid (TKG) ecosystem.

🔹 TKG – Tanzu Kubernetes Grid

Definition:
VMware’s enterprise Kubernetes platform to deploy, operate, and manage Kubernetes clusters across vSphere, AWS, Azure, and other infrastructures.

✅ Key Features:

  • Unified CLI (tanzu)
  • Declarative cluster management using Cluster API
  • Multi-cloud support
  • Secure, tested Kubernetes releases (TKRs)
  • Supports both Standalone and vSphere with Tanzu (TKGS) modes

🧱 TKG Deployment Modes:

Mode Description
TKG Standalone Uses a Management Cluster to manage lifecycle of Workload Clusters
TKGS (vSphere with Tanzu) Uses a Supervisor Cluster embedded in vSphere to manage TKCs (Guest Clusters)

🔹 TKR – Tanzu Kubernetes Release

Definition:
A VMware-curated Kubernetes distribution that includes:

  • A specific Kubernetes version
  • Core system images (e.g., CoreDNS, kube-proxy)
  • Add-ons (CNI, CSI, metrics-server, etc.)
  • BOM file listing component versions

📦 Example TKR Name:

v1.26.8---vmware.1-tkg.1

🛠️ TKR Usage:

  • In TKG Standalone: referenced during workload cluster creation
  • In TKGS (vSphere with Tanzu): used in TanzuKubernetesCluster specs

🔹 TKC – Tanzu Kubernetes Cluster

Definition:
A Kubernetes Custom Resource (kind: TanzuKubernetesCluster) used in TKGS to declaratively define Kubernetes clusters inside a vSphere Namespace.

📄 Example TKC YAML:

apiVersion: run.tanzu.vmware.com/v1alpha1
kind: TanzuKubernetesCluster
metadata:
  name: dev-cluster
  namespace: dev-ns
spec:
  distribution:
    version: v1.26.8---vmware.1-tkg.1
  topology:
    controlPlane:
      count: 1
      class: best-effort-small
    workers:
      count: 3
      class: best-effort-small

📝 Used only in vSphere with Tanzu (TKGS), not TKG Standalone

🔹 Supervisor Cluster

Definition:
A Kubernetes control plane embedded directly into vSphere when you enable Workload Management.

✅ Key Roles:

  • Hosts Spherelet (Kubelet for ESXi)
  • Schedules vSphere Pods (optional)
  • Manages vSphere Namespaces
  • Deploys and manages Tanzu Kubernetes Clusters (TKCs)

📌 Runs on:

  • ESXi hosts directly (not inside VMs)
  • Created and managed by vCenter

🛠️ Enabled By:

  • Going to vCenter > Workload Management
  • Selecting compatible networking (NSX-T, HAProxy, AVI)

🔹 Management Cluster (TKG Standalone)

Definition:
A Kubernetes cluster created by TKG CLI to bootstrap and manage other (workload) clusters.

✅ Functions:

  • Hosts Cluster API controllers
  • Manages lifecycle of workload clusters
  • Used in TKG on vSphere, AWS, Azure

📌 Created via:

tanzu management-cluster create

🔹 Workload Cluster

Definition:
A standard Kubernetes cluster deployed by the Management Cluster (TKG Standalone) to run your actual applications.

✅ Characteristics:

  • Uses a selected TKR
  • Runs on VMs provisioned by the management cluster
  • Can be scaled, upgraded, or deleted using tanzu CLI

📌 Created via:

tanzu cluster create my-cluster --tkr <version>

🔁 Cluster Relationships Overview

TKG (Platform)
├── Uses TKR (versioned Kubernetes releases)
│
├── In Standalone mode:
│     ├── Management Cluster (bootstrap)
│     └── Workload Clusters (app workloads)
│
└── In vSphere with Tanzu (TKGS):
      ├── Supervisor Cluster (on ESXi)
      └── Tanzu Kubernetes Clusters (TKCs, created as CRs)

🧭 Summary Table of Cluster Types

Cluster Type Where It’s Used Description
Supervisor Cluster TKGS (vSphere with Tanzu) Native Kubernetes layer running on ESXi
TKC TKGS Custom Resource to create workload clusters
Management Cluster TKG Standalone Manages lifecycle of other clusters
Workload Cluster TKG Standalone Runs your applications; managed by Management Cluster

✅ Summary of Core Terms

Term Full Form What It Is Used In Example
TKG Tanzu Kubernetes Grid Full Kubernetes platform by VMware Standalone, TKGS tanzu cluster create ...
TKR Tanzu Kubernetes Release Curated Kubernetes version and components TKG & TKGS v1.26.8---vmware.1-tkg.1
TKC Tanzu Kubernetes Cluster CR to deploy a K8s cluster in TKGS Only in TKGS kind: TanzuKubernetesCluster
Supervisor Cluster – Embedded Kubernetes in vSphere Only in TKGS Runs on ESXi directly
Management Cluster – Cluster that manages workload clusters Only in TKG Standalone Created via CLI
Workload Cluster – K8s cluster for app workloads Only in TKG Standalone Created by Management Cluster
